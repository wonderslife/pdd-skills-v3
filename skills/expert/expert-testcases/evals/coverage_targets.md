# 覆盖率目标定义 (Coverage Targets)

> **版本**: v1.0
> **适用范围**: expert-testcases 生成的测试用例集

---

## 一、覆盖率维度与目标

### 1.1 API端点覆盖

| 指标 | 目标值 | 计算方式 | 优先级 |
|------|--------|---------|--------|
| 端点覆盖率 | **100%** | 已测试端点数 / 总端点数 | P0 |
| HTTP方法覆盖 | **100%** | 已覆盖方法集 / {GET,POST,PUT,DELETE} | P0 |
| 权限注解覆盖 | **100%** | 已验证权限数 / @PreAuthorize总数 | P0 |

**验证方法**：
```bash
python scripts/extract_api.py --project <path> --module <name> --output api.json
# 对比 api.json 中的端点与测试用例中的 API_ID
```

### 1.2 权限场景覆盖

| 指标 | 目标值 | 计算方式 | 优先级 |
|------|--------|---------|--------|
| 正向权限覆盖 | **100%** | 有权限角色测试数 / 有权限角色总数 | P0 |
| 反向权限覆盖 | **≥90%** | 无权限角色测试数 / 无权限角色总数 | P1 |
| 越权场景覆盖 | **≥80%** | 越权测试数 / 越权场景总数 | P1 |
| 数据权限覆盖 | **≥90%** | @DataScope测试数 / @DataScope注解数 | P0 |

**角色×操作矩阵示例**：

```
               申请人  子公司审批  集团审批  管理员  跨部门用户
A01-列表查询    ✅      ✅       ✅      ✅      ❌(跨部门)
A02-查看详情    ✅      ✅       ✅      ✅      ❌(跨部门)
A03-新增申请    ✅      ❌       ❌      ✅      ❌(无权限)
A09-审批操作    ❌      ✅       ✅      ✅      ❌(无权限)
```

### 1.3 状态转移覆盖

| 指标 | 目标值 | 计算方式 | 优先级 |
|------|--------|---------|--------|
| 状态码覆盖 | **100%** | 已测试状态数 / 总状态数 | P0 |
| 关键转移覆盖 | **100%** | 已覆盖关键转移数 / 关键转移总数 | P0 |
| 全转移覆盖 | **≥90%** | 已覆盖转移数 / 合法转移总数 | P1 |

**关键转移路径**（必须100%覆盖）：

```
01(待提交) → 02(已提交)      # 提交申请
02(已提交) → 03(子公司通过)   # 子公司审批通过
02(已提交) → 08(已驳回)      # 子公司审批驳回
08(已驳回) → 02(已提交)      # 驳回重提
03(子公司通过) → 04(集团通过) # 集团审批通过
04(集团通过) → 07(决议通过)   # 决议上传通过
01(待提交) → 09(已终止)      # 终止申请
```

### 1.4 代码覆盖

| 指标 | 目标值 | 工具 | 优先级 |
|------|--------|------|--------|
| 行覆盖率 (Line) | **≥80%** | pytest-cov / Jacoco | P1 |
| 分支覆盖率 (Branch) | **≥70%** | pytest-cov / Jacoco | P1 |
| 函数覆盖率 (Function) | **≥85%** | pytest-cov / Jacoco | P1 |

**重点覆盖的代码区域**：
- ServiceImpl 中的审批处理方法（handleApproval）
- ServiceImpl 中的状态变更逻辑
- Controller 中的参数校验逻辑
- Mapper XML 中的数据权限过滤

### 1.5 场景覆盖

| 场景类型 | 目标值 | 优先级 |
|---------|--------|--------|
| Happy Path | **100%** | P0 |
| Rejection Path | **100%** | P0 |
| Termination Path | **100%** | P1 |
| 并发竞态 | **≥60%** | P2 |
| 数据权限隔离 | **100%** | P0 |

## 二、各层级最低用例数

| 层级 | 最低用例数 | 计算公式 | 说明 |
|------|-----------|---------|------|
| API层 | 端点数 × 3 | 每端点至少：正向+权限+异常 | P0 |
| 集成层 | 工作流节点数 × 2 | 每节点：通过+驳回 | P0 |
| UI层 | 页面数 × 4 | 每页面：渲染+交互+验证+异常 | P1 |
| E2E层 | 3 | Happy Path + Rejection + Termination | P0 |
| 边界异常层 | 端点数 × 1.5 | null/越界/格式/权限/并发 | P1 |
| 性能层 | 关键端点数 | 列表查询+详情+并发+导出 | P2 |

## 四、覆盖率提升策略

当覆盖率未达标时，按以下优先级补充：

1. **P0 缺口**：立即补充，阻塞发布
   - 缺失的API端点测试
   - 缺失的关键状态转移测试
   - 缺失的数据权限测试

2. **P1 缺口**：本迭代补充
   - 权限反向测试（无权限/越权）
   - 非关键状态转移
   - 代码分支覆盖

3. **P2 缺口**：下迭代补充
   - 并发竞态场景
   - 性能基线测试
   - UI自动化测试
