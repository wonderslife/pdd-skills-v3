# 测试用例质量评分标准 (Quality Rubric)

> **版本**: v1.0
> **适用范围**: expert-testcases 生成的所有测试用例

---

## 一、评分维度

### 1.1 完整性 (Completeness) - 权重 30%

| 等级 | 分数 | 标准 |
|------|------|------|
| A | 90-100 | 所有必填字段完整填写，无遗漏；用例覆盖所有API端点、状态码、角色 |
| B | 75-89 | 必填字段完整，个别可选字段缺失；覆盖率≥95% |
| C | 60-74 | 少量必填字段缺失；覆盖率≥85% |
| D | <60 | 大量字段缺失；覆盖率<85% |

**检查项**：
- [ ] 每条用例包含：ID、目的、优先级、类型、前置条件、操作步骤、预期结果
- [ ] API端点覆盖率 = 100%
- [ ] 状态码覆盖率 = 100%
- [ ] 角色权限矩阵完整（5角色 × N操作）
- [ ] test-data.yaml 包含5个基础角色

### 1.2 准确性 (Accuracy) - 权重 25%

| 等级 | 分数 | 标准 |
|------|------|------|
| A | 90-100 | 预期结果与实际系统行为完全一致；API路径/参数无错误 |
| B | 75-89 | 预期结果基本正确，少量需微调；API路径正确 |
| C | 60-74 | 部分预期结果不准确，需人工修正 |
| D | <60 | 大量预期结果错误，需重写 |

**检查项**：
- [ ] API路径与Controller实际路径一致
- [ ] 权限标识与@PreAuthorize注解一致
- [ ] 状态码与枚举定义一致
- [ ] 表单字段与Entity定义一致
- [ ] 工作流节点与BPMN定义一致

### 1.3 可运行性 (Runnability) - 权重 20%

| 等级 | 分数 | 标准 |
|------|------|------|
| A | 90-100 | pytest --collect-only 通过率100%；py_compile通过率100% |
| B | 75-89 | collect通过率≥95%；py_compile通过率100% |
| C | 60-74 | collect通过率≥85%；少量语法错误 |
| D | <60 | 大量语法错误，无法收集 |

**检查项**：
- [ ] 所有.py文件 py_compile 通过
- [ ] pytest --collect-only 收集数 = 文档用例数 ±5%
- [ ] conftest.py fixture 可正常加载
- [ ] config.py 配置项完整
- [ ] requirements.txt 依赖完整

### 1.4 可维护性 (Maintainability) - 权重 15%

| 等级 | 分数 | 标准 |
|------|------|------|
| A | 90-100 | 测试数据与逻辑完全分离；ID命名规范统一；fixture复用率高 |
| B | 75-89 | 数据基本分离；命名规范；少量硬编码 |
| C | 60-74 | 部分数据硬编码；命名基本规范 |
| D | <60 | 大量硬编码；命名混乱 |

**检查项**：
- [ ] 测试数据从test-data.yaml加载，无硬编码
- [ ] 用例ID遵循 TC-{CODE}-{LEVEL}{SEQ}-{name} 规范
- [ ] pytest marker 使用规范（p0/p1/p2, functional/permission等）
- [ ] fixture复用（session scope, 数据fixture）
- [ ] ApiHelper封装统一

### 1.5 覆盖深度 (Depth) - 权重 10%

| 等级 | 分数 | 标准 |
|------|------|------|
| A | 90-100 | 6层测试全覆盖；含并发/竞态/数据权限场景 |
| B | 75-89 | 4层以上覆盖；含权限和异常场景 |
| C | 60-74 | 2-3层覆盖；仅正常路径 |
| D | <60 | 仅1层覆盖；仅Happy Path |

**检查项**：
- [ ] API层（Level 1）用例数 ≥ 端点数 × 3
- [ ] 集成层（Level 2）覆盖工作流全链路
- [ ] 边界异常层（Level 5）覆盖null/越界/格式/权限/并发
- [ ] E2E层（Level 4）覆盖Happy Path + Rejection + Termination
- [ ] 数据权限测试覆盖同部门/跨部门/越权

---

## 二、综合评分

```
总分 = 完整性×0.30 + 准确性×0.25 + 可运行性×0.20 + 可维护性×0.15 + 覆盖深度×0.10
```

| 总分 | 等级 | 判定 |
|------|------|------|
| 90-100 | A | 优秀 - 可直接使用 |
| 75-89 | B | 良好 - 微调后可用 |
| 60-74 | C | 合格 - 需人工修正 |
| <60 | D | 不合格 - 需重新生成 |

---

## 三、自动化检查脚本

```bash
# 完整性检查
python scripts/validate_output.py --output-dir <path> --verbose

# 可运行性检查
python -m py_compile <path>/**/*.py
pytest --collect-only <path>/01-api-testcases/

# 覆盖率检查
python scripts/validate_output.py --output-dir <path> --run-pytest
```

---

## 四、评分流程

1. **自动评分**：validate_output.py 输出 stats 和 errors/warnings
2. **人工抽检**：随机抽取10%用例验证准确性
3. **运行验证**：在测试环境执行，记录通过率
4. **综合评定**：加权计算总分，确定等级
