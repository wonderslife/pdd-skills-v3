# 覆盖率最佳实践

## 6维覆盖率指标体系

### 1. API端点覆盖率

**目标**: 100%
**计算**: 已覆盖端点数 / 总端点数
**保障方法**: ApiExtractor自动扫描 → 与用例API_ID比对 → 缺失端点自动补全

### 2. 权限场景覆盖率

**目标**: ≥95%
**计算**: 已覆盖权限场景数 / 总权限场景数
**保障方法**: 权限矩阵 × 角色 → 每个✅至少1个正向用例，每个❌至少1个403用例

### 3. 状态转移覆盖率

**目标**: ≥90%
**计算**: 已覆盖转移数 / 合法转移总数
**保障方法**: 状态机分析 → 每条转移至少1个用例

### 4. 代码行覆盖率

**目标**: ≥80%
**工具**: pytest-cov (Python) / Jacoco (Java)
**命令**: `pytest --cov --cov-report=html`

### 5. 分支覆盖率

**目标**: ≥70%
**重点**: ServiceImpl中的if/else/switch分支

### 6. 函数覆盖率

**目标**: ≥85%
**重点**: Service接口的所有public方法

## 5大提升策略

### 策略1: 端点驱动
1. ApiExtractor扫描所有@RequestMapping
2. 生成端点清单
3. 与用例API_ID比对
4. 缺失端点 → 自动补全最小用例(200校验)

### 策略2: 权限矩阵驱动
1. 构建 角色 × 操作 矩阵
2. 标记✅(有权限)/❌(无权限)/⚠️(越权)
3. ✅至少1个正向用例, ❌至少1个403用例

### 策略3: 状态机驱动
1. 提取状态码集合
2. 构建合法转移图
3. 每条转移至少1个用例覆盖
4. 重点: 01→02(提交), 02→08(驳回), 08→02(重提)

### 策略4: 源码分支驱动
1. 分析ServiceImpl中的if/else/switch
2. 每个分支至少1个用例触达
3. 关键: handleApproval中的usertask2~6分支

### 策略5: 缺陷反向驱动
1. 收集该模块历史bug清单
2. 每个bug生成对应回归用例
3. 标记regression marker
