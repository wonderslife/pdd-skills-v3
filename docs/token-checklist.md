# Token Efficiency Checklist - Token效率检查清单

> 本清单用于评估和优化技能内容的Token消耗。
> 目标：在不损失信息质量的前提下最小化Token使用。

---

## Token效率检查项

### 1. 元数据精简 (Layer 0)

| 检查项 | 标准 | 权重 |
|-------|------|------|
| frontmatter是否只保留必要字段 | 只保留 name, description, version | 高 |
| description是否控制在200字符以内 | 超长描述增加每次加载成本 | 高 |
| 是否移除了冗余的metadata字段 | 如空的 author, license 等 | 中 |

### 2. 内容分层 (Layer 1/2/3)

| 层级 | 内容 | 加载时机 | Token预算 |
|------|------|---------|----------|
| Layer 0 | 名称+描述+触发词 | 始终加载 | <500 tokens |
| Layer 1 | 核心方法论+流程 | 技能被选中时 | <2000 tokens |
| Layer 2 | 详细步骤+示例 | 执行对应步骤时 | <5000 tokens/步 |
| Layer 3 | 参考文档+完整规范 | 用户显式请求时 | 按需加载 |

### 3. 文本密度优化

| 检查项 | 优化策略 | 预估节省 |
|-------|---------|---------|
| 过长的表格转为紧凑格式 | 用缩进列表替代宽表格 | 30-50% |
| 重复的内容提取为引用 | 定义一次，多处引用 | 20-40% |
| 冗余的解释性文字 | 保留关键决策点，删除过程描述 | 20-30% |
| ASCII图表简化 | 精简为必要节点和连接线 | 40-60% |
| 多语言重复内容 | 使用条件引用而非并列 | 25-35% |

### 4. 结构优化

| 检查项 | 标准 |
|-------|------|
| 单个文件总Token数 | 建议 <8000 tokens (约4000中文字) |
| 单个章节Token数 | 建议 <1500 tokens |
| 代码示例长度 | 每个<100行，超过则引用外部文件 |
| 版本历史长度 | 只保留最近3个版本 |

### 5. 触发词精准度

| 检查项 | 标准 |
|-------|------|
| 触发词与功能的匹配度 | 避免过于宽泛的触发词 |
| 触发词去重 | 同义词合并，减少误触发 |
| 中英文触发词覆盖 | 各至少3个精准触发词 |

---

## Token计算公式

```
EffectiveTokens = BaseTokens × (1 - LayerSavings) × (1 - DensityGain)

其中:
- BaseTokens: 原始文件token数
- LayerSavings: 通过分层节省的比例 (目标: 0.3~0.5)
- DensityGain: 通过文本密度优化提升的效率 (目标: 0.15~0.25)
```

## 如何使用

```bash
npx pdd-skills cso --token-analyze
npx pdd-skills cso --token-optimize <skill-name>
```
